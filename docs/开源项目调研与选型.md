# 商机洞察 Agent — 开源项目调研与选型

> 调研日期 2026-06-04。目标：为中小企业商机洞察 Agent 寻找可借鉴/可复用的 GitHub 开源项目。
> star 数为调研时快照量级，以仓库实时数据为准。

## 一句话结论

GitHub 上**没有**开箱即用、覆盖本项目 [商机洞察流程示意.pdf](商机洞察流程示意.pdf) 全流程的成熟选品 Agent。
现状是「通用研究 Agent 框架成熟 + 数据采集组件成熟，但电商洞察的编排与打分逻辑需自建」。
**最优策略 = 分层组装**：LangGraph 研究骨架（流水线引擎） + TikHub（数据源） + 自建电商洞察 prompt 层（核心价值）。

## 选定路线（已决策）

- **流水线骨架**：基于 **LangGraph 自建**，参照 `company-research-agent` 的范式
- **数据源**：继续走 **TikHub 商用 API**（合规、稳定、TikTok Shop 数据全），延续现有 `采集工作台/scripts` 探测脚本

## PDF 流程 → 架构映射

| PDF Step | 做什么 | 对应 LangGraph 节点 | 数据来源 |
|---|---|---|---|
| Step 1 关键词分析 | 叶子类目 → AI 识别 10-20 精准词 | Planner / KeywordExtractor | 数仓精准词表 + LLM |
| Step 2 社媒分析 | persona + KOC/KOL 归类 + 需求维度矩阵 | SocialAnalyzer（并行） | 小红书账号/笔记/评论（TikHub） |
| Step 3 价格带分析 | TOP300 价格带诊断 + 机会带 | PriceAnalyzer（并行） | 商品搜索（TikHub） |
| Step 4 热销链接分析 | 爆款元素 + 卖点 + 主图视觉解构 | HotItemAnalyzer（并行） | 商品链接+主图（TikHub） |
| Step 5 竞品详情分析 | 结构化对比 + 评价词云 + 正负向洞察 | CompetitorAnalyzer（并行） | 评价/问大家（TikHub 评论端点） |
| → 报告 Agent | 人群/场景/价格/卖点/视觉/风险 综合 | Curator → Briefing → Editor | 全流程汇总 |

> 「精准词」贯穿全流程：Step 1 产出后作为 Step 2-5 的核心筛选条件，需作为 LangGraph 全局 state 透传。

- 情感/词云起步：`snownlp`（语料即电商评论）+ `jieba` + `wordcloud` → 直接出 Step 5 评价词云

## 可直接复用的架构模式（跨项目共识）

1. **规划-执行分离**：Planner 先把类目拆成子问题/大纲，再并行执行
2. **并行 Researcher + 顺序 Processing**：多维并行抓取 → Collector 聚合 → Curator 相关性打分过滤 → Briefing 分类摘要 → Editor 终稿
3. **双模型分工**：长上下文综合用便宜大窗口模型，精确格式化用强指令模型（省钱已验证）
4. **审校回环**：Reviewer→Revisor 保障洞察可靠性
5. **配置化报告模板**：task.json 式控制 sections/导出格式
6. **多视角提问**：STORM perspective 提升调研深度

## 已克隆到本地的参考仓库（reference/，已 gitignore）

### 1. company-research-agent —— 真代码，骨架可直接抄 ✅

`reference/company-research-agent/backend/graph.py` 是干净的 LangGraph 编排，结构与本项目 PDF 几乎同构：

```
grounding（入口预处理）
  → 并行 4 Analyzer: financial / news / industry / company
  → collector（聚合）→ curator（相关性打分过滤）
  → enricher → briefing（分类摘要）→ editor（出报告）
```

**映射到本项目**：4 Analyzer → 社媒/价格带/热销/竞品；grounding → Step1 关键词分析；数据源 Tavily → TikHub；editor 出报告。
- `backend/classes/state.py` 的 `ResearchState`：「原始数据字段 + curated 后字段」并存的透传范式，正好用来透传精准词与各维度数据
- `backend/nodes/curator.py`：相关性打分过滤（阈值过滤）可借鉴到选品打分
- `backend/nodes/briefing.py` / `editor.py`：双模型 prompt 工程

### 2. eCommerce-Skills —— ⚠️ 是 prompt 话术，不是可复用代码

读 SKILL.md 后确认：**每个 skill 都是同一模板套出的 56-63 行薄壳**，含给 LLM 的 best-practices 提示词 + 引流到商业产品 Nexscope 的广告，**无任何采集代码 / 打分算法 / API 接入**（官方原话 "not direct TikTok API integration"）。

唯一价值：提炼其**分析框架与输出格式规范**，当各分析节点的 prompt 蓝本。与本项目 5 步对口的 skill：

| PDF Step | 对口 skill | 可提炼什么 |
|---|---|---|
| Step 1 关键词 | `ecommerce-keyword-research` | 关键词分类/筛选维度 |
| Step 2 社媒 | `tiktok-influencer-marketing`、`social-media-monitor` | KOL/KOC 评估维度 |
| Step 3 价格带 | `competitor-price-analysis`、`tiktok-shop-pricing` | 价格带诊断维度 |
| Step 4 热销 | `tiktok-shop-trending-products`、`tiktok-shop-product-research` | 爆款元素清单 |
| Step 5 竞品 | `product-review-analysis`、`customer-feedback-analysis`、`ecommerce-competitor-analysis`、`market-gap-analysis` | 痛点分类/情感趋势/正负向洞察的输出格式 |

> 用法：人工读对口 skill 的「Capabilities / Output Format」段落，把维度清单抄进对应 LangGraph 节点的 system prompt。不要试图把 skill 当代码 import。
