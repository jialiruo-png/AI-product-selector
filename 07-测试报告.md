## 三、测试中发现并修复的 3 个真实数据 bug

这三个都是**只有接真实数据才会暴露**的坑，mock 模式发现不了：

1. **价格段空白**
   `rank_products` 重塑商品后把真实价格放到**顶层扁平字段**、`raw.price` 置为 `None`，而下游一律读 `raw` → 价格全 0。
   修复：新增 `_pick()` 三级回退（顶层 → raw → 嵌套 `product_price_info.sale_price_decimal`），价格 / 销量 / 评分 / 图片全部修正。

2. **痛点抽取静默失败（缺 key）**
   `AnalyzeReviewsTool` 没加载项目根 `.env` → 取不到 LLM key → 抛错。
   修复：在 `import analyze` 前补 `_load_client()`（它负责 `load_env`）。

3. **DeepSeek 思考模式 token 截断（最隐蔽）**
   `analyze.py` 写死 `max_tokens=2048`，而 DeepSeek V4 思考模式（pro / flash 都默认开思考）的**思考过程吃光 2048 额度** → JSON 没输出完就被截断 → 痛点全空，但 `ok=True`（静默失败）。
   修复：JSON 抽取类任务改用**非思考模型 `deepseek-chat`**（实测 ~378 token 即出完整 JSON），通过 `.env` 的 `LLM_MODEL_EXTRACT` 配置，工具层临时覆盖 model。
   > 诊断信号：`_stats.completion_tokens` 顶到 2048 且 pain_points 为空 = 被截断。

## 四、模型分工（已落地到 .env）

| 任务类型 | 模型 | 原因 |
|---|---|---|
| 生成类（精准词 / 摘要 / 报告） | `deepseek-v4-pro`（思考模式） | 提质 |
| JSON 抽取类（评论痛点） | `deepseek-chat`（非思考） | 思考模式撞 2048 截断，非思考稳且省 |

> 注：原本想「抽取用 flash」，但实测 `deepseek-v4-flash` **仍是思考模式、照样撞 2048**，真正不截断的是非思考的 `deepseek-chat`。
